## 2. 前后端同步规则

从 04 Sub2API 集成开始，不能再只交付后端接口。每个涉及管理端使用的模块，必须同时完成最小可见页面、接口封装、菜单入口和构建检查。

当前同步要求:

1. 11 前端基础已补齐。
2. 04 Sub2API 集成前端入口已回补。
3. 05 强一致调额已完成后端 API、基础前端页面、提交确认文案、真实前后端联通验收和 H5 调额流程验收。
4. 后续模块继续按“后端 API + 前端页面 + 测试/构建 + 验收记录 + 文档进度”一起验收。

模块完成定义:

- 后端接口完成但没有前端入口，只能算“后端完成”，不能算模块完成。
- 前端页面没有接真实 API，只能算“页面骨架完成”，不能算模块完成。
- 调额、记账、对账类模块必须后端和前端都验收通过后，才能在 HTML 看板勾选模块完成项。
- 后续所有业务模块必须有验收记录，至少包含桌面端、H5 手机端、前后端真实联通、异常路径。
- `php artisan test`、`pnpm build` 只能证明代码可运行，不能替代业务验收。
- `progress.html` 里每个后续模块必须拆出验收项，验收未完成时不能把模块勾到 100%。
