# 模块进度模板

复制本模板创建新模块进度文件。

```markdown
# 模块名

状态: 未开始

## 1. 目标

- 写清楚这个模块解决什么问题。

## 2. 范围

- 包含什么。
- 不包含什么。

## 3. 依赖

- 上游模块。
- 外部服务。

## 4. 交付物

- 后端文件。
- 前端文件。
- 测试文件。
- 文档。

## 5. 进度

| 项 | 状态 | 备注 |
|---|---|---|
| 设计 | 未开始 |  |
| 后端 | 未开始 |  |
| 前端 | 未开始 |  |
| 测试 | 未开始 |  |
| 桌面端验收 | 未开始 |  |
| H5 手机端验收 | 未开始 |  |
| 前后端联通验收 | 未开始 |  |
| 异常路径验收 | 未开始 |  |
| 文档 | 未开始 |  |

## 6. 验收标准

- 明确可验证结果。
- 必须写清楚桌面端验收结果。
- 必须写清楚 H5 手机端验收结果。
- 必须写清楚前后端真实 API 联通结果。
- 必须写清楚异常路径如何验证。
- 只有测试、构建通过但没有验收记录时，模块不能标记为已完成。

命令
