# 开发与联调检查清单

## 本地启动

```bash
cd backend
php artisan migrate --seed
php artisan serve --host=127.0.0.1 --port=8010

cd ../frontend
VITE_API_PROXY_TARGET=http://127.0.0.1:8010 pnpm dev --host 127.0.0.1 --port 5174
```

默认本地管理员:

```text
admin@example.com / admin123
```

## 必跑检查

前端命令需要 Node >=20.19 或 >=22.12。当前本机默认 `/usr/local/bin/node` 是 v14 时，需先切到新版本再跑 `pnpm`。

```bash
cd backend
php artisan test

cd ../frontend
pnpm typecheck
pnpm build
pnpm e2e
```

## 桌面端验收

- 登录后能进入首页看板。
- Sub2API 用户、模型统计、用户与额度、调额记录、现金/赠送账、经营账、对账中心、异常中心、操作审计都能打开。
- 调额提交前必须出现强确认文案。
- 成功记录页只显示 `succeeded` 且已二次确认的调额单。
- 经营账详情可查看安全富文本和附件。
- 对账差异非 0 时状态为有差异。

## H5 手机端验收

- 390px 左右宽度可登录。
- 手机端菜单可打开并切换业务页面。
- 筛选区不遮挡按钮。
- 表格允许横向滚动，金额和状态字段可查看。
- 弹窗、抽屉可正常关闭。

## 异常路径验收

- 未登录访问 API 返回 401。
- Sub2API 调额失败不会显示成功。
- 调额二次确认失败进入异常中心。
- 现金金额和赠送额度之和不等于调额额度时返回 422。
- 非法附件类型返回失败。
- 富文本脚本保存后被过滤。
- 重复生成同一天对账批次返回 409。

## 真实调额验收

真实 Sub2API 调额必须使用小额测试用户。

1. 记录测试用户当前 Sub2API 余额。
2. 在新系统提交小额增加额度。
3. 等待新系统返回 `Sub2API 已入账并确认成功`。
4. 在 Sub2API 查询同一用户余额，确认真实增加。
5. 检查额度调整记录、现金账/赠送账、对账中心、操作审计。
6. 如 Sub2API 返回失败或确认余额不一致，新系统不能显示成功。
