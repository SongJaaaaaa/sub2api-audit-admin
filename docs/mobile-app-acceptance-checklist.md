## 2. 发布前环境

- [ ] `VITE_APP_MODE=app` 已用于 App 构建。
- [ ] `VITE_API_BASE_URL` 是正式 HTTPS 绝对地址，不是相对 `/api/v1`，也不是开发代理地址。
- [ ] 构建产物没有 API Key、数据库密码、签名密码或 Token。
- [ ] `CORS_ALLOWED_ORIGINS` 使用明确逗号列表，没有 `*`。
- [ ] CORS 来源包含实际 Capacitor scheme/hostname，且允许 `Authorization`、`Content-Type`、`Accept` 和 `OPTIONS`。
- [ ] 已验证登录、`/auth/me`、401、附件上传、附件下载和导出请求。
- [ ] Android 签名材料在仓库外；iOS 证书和 Provisioning Profile 在受控账号/钥匙串中。
- [ ] 构建版本号、提交号、产物哈希已记录。

## 3. 安装与生命周期

- [ ] 首次安装后 App 能启动到登录页。
- [ ] 覆盖升级后 App 能启动，既有安全 Token 按预期迁移或要求重新登录。
- [ ] 卸载重装不会错误恢复旧 Token。
- [ ] App 切后台再恢复后页面和请求状态正常。
- [ ] 旋转屏幕或改变系统字体后没有关键内容遮挡。
- [ ] 刘海、状态栏和底部手势区不会遮住标题、表单或提交按钮。
- [ ] Android 系统返回键按顺序关闭弹层、返回详情、返回模块首页，再执行退出确认。
- [ ] iOS 边缘返回和顶部返回按钮行为一致。

## 4. 登录与安全

- [ ] 正确账号密码登录后进入 App 模块首页，不出现桌面侧边栏、抽屉或汉堡菜单。
- [ ] 错误账号密码显示后端业务错误，不显示假成功。
- [ ] 冷启动读取安全存储并调用 `/auth/me`。
- [ ] Token 失效收到 401 后清理本地会话并回到登录页。
- [ ] 主动退出先调用服务端注销，再清理本地会话。
- [ ] 日志、网络调试记录和截图中没有 Token、Authorization 或密码。
- [ ] 无网络时显示明确错误和重试入口，不伪造缓存成功结果。

## 5. 首页与导航

- [ ] 首页直接显示所有正式模块，使用单层两列宫格，不显示业务分类中间页。
- [ ] 模块项有图标、名称、可靠状态/数字和足够触控面积。
- [ ] 每个模块最多一次点击进入正确业务路由。
- [ ] 模块页标题、返回按钮和主要操作固定在安全区域内。
- [ ] 从详情返回列表时筛选条件、滚动位置和已加载页数保持正确。
- [ ] 有未提交表单时离开页面会二次确认。

## 6. 移动布局与通用列表

在 320x568、375x812、390x844、430x932 和 768x1024 视口各检查一次：

- [ ] 页面没有非预期横向滚动。
- [ ] 列表使用摘要卡片完成主要操作，不依赖宽表格横向滚动。
- [ ] 卡片展示主体、状态、关键金额/指标和时间等 3 至 5 个字段。
- [ ] 长邮箱、模型名和备注最多两行并正确省略，未撑破卡片。
- [ ] 搜索框、筛选按钮、生效条件标签和结果数量清晰可见。
- [ ] 筛选面板底部的“重置”和“查询”在键盘弹出时仍可点击。
- [ ] 加载、空数据、失败、刷新和没有更多数据状态明确。
- [ ] “加载更多”不会重复请求、漏项或清空已有数据。
- [ ] 详情页能查看桌面端完整字段，危险操作不直接暴露在卡片表面。
- [ ] 底部操作栏位于安全区上方，提交期间禁用重复点击。

## 7. P0 业务流程

### Sub2API 用户

- [ ] 关键词查询、状态筛选和余额排序结果与桌面一致。
- [ ] 用户卡片进入详情后可查看完整信息和历史记录。
- [ ] 批量赠送能选择对象、显示进度，并分别展示成功数和失败数。
- [ ] 金额操作提交前显示对象、金额、操作类型和影响结果确认摘要。

### 用户充值

- [ ] 充值表单为独立页面或全屏容器，键盘不会遮挡输入和提交按钮。
- [ ] 成功后余额、详情、列表和汇总同步刷新。
- [ ] 失败、401、422、409 和 502 显示真实错误；重复点击不会重复入账。

### 异常中心

- [ ] 风险卡片显示用户、原因、状态和发生时间。
- [ ] 查看、重试和作废操作有明确确认；重试失败不会伪造成功。
- [ ] 列表和汇总使用同一筛选条件。

## 8. P1/P2 业务流程

- [ ] 收入：筛选、详情、新增和列表刷新正确。
- [ ] 支出：分类、金额、日期、新增和列表刷新正确。
- [ ] 历史账：筛选、详情、分页和导出正确。
- [ ] 利润统计：周期切换、图表、确认和撤销金额摘要正确。
- [ ] 数据总览：KPI、趋势和排行在小屏无空白、裁切或重叠。
- [ ] 消费排行：模型名、邮箱、Token、请求数和金额可完整查看。
- [ ] 操作审计：详情按字段分组，管理员和对象信息正确。
- [ ] 管理员账号：姓名、邮箱、状态和创建时间正确。

## 9. 原生能力

- [ ] 状态栏颜色和图标对浅色/深色主题均有足够对比度。
- [ ] 键盘弹出时页面可滚动，固定提交栏不被遮挡。
- [ ] 断网显示明确状态；恢复网络后手动重试成功。
- [ ] 文件选择、图片/PDF 上传成功，类型和大小限制与 Web 一致。
- [ ] 文件下载能保存到系统文件或调用系统分享/打开能力。
- [ ] 外部链接交给系统浏览器，不在管理 WebView 内任意跳转。
- [ ] Android 返回键和 iOS 返回手势不会重复提交或跳错页面。

## 10. 桌面回归与性能

- [ ] 桌面 Chrome 仍显示侧边栏、表格、列设置和原有详情抽屉。
- [ ] 桌面登录、查询、筛选、分页、写操作和导出无回归。
- [ ] App 首页只请求必要数据，没有为每个模块重复加载统计接口。
- [ ] 长列表加载期间页面仍可滚动和点击。
- [ ] 图表页面离开后释放监听和实例，返回页面不会重复绘制。
- [ ] 冷启动到首个可操作页面的耗时已记录：`______ ms`。
